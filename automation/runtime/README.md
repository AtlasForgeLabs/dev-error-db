# Automation Runtime Files

This directory is for local and CI-generated pipeline runtime state.

Do not commit concrete runtime JSON files from this directory. Files such as `pipeline-state.json`, `pipeline-health.json`, `pipeline-validation.json`, and `seo-health.json` are generated by automation commands and represent the state of a specific run.

The automation scripts create the required JSON files when they are missing:

- `npm run report` writes `seo-health.json`.
- `npm run pipeline:validate` writes `pipeline-validation.json` and updates health output.
- `npm run pipeline:health` writes `pipeline-health.json`.
- Guardrail-aware write flows create `pipeline-state.json` when they need publish safety state.

Commit this README and `.gitkeep` so the directory exists, but keep runtime JSON output out of Git.
